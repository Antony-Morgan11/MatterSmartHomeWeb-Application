    About

    Architecture

      • The diagram below describes the underlying architecture behind our project The devices and services we will use for this project are Javascript for frontend, Flask, and SQLAlchemy for backend and database, Raspberry Pi (as the bridge/gateway to the Matter network), and lastly, nRF5340 DK Nordic board.

      • The computer on the far left of the diagram represents our web application and is the first layer of our project. As described earlier, this is the layer where the client can manipulate their Matter light bulb and switch.

      • We will set up a Flask server on the backend to process requests to the matter devices as it will also authenticate users and authorize sessions. When the flask server receives requests destined for the matter device, it will forward those requests to a matter controller “Linux CHIP-Tool”; which handles the matter thread network.

      • Our Raspberry Pi will act as the “Bridge/Gateway” to communicate with the matter network. A bridge device is a device that connects two different networks with each other; in this case, it allows our Wi-Fi network to communicate with our thread network. A bridge device is necessary in our case because our end device (the nRF5340 DK) communicates over Open Thread 1.3 protocol and cannot communicate over Wi-Fi. To solve this, we will use a Raspberry Pi as the OTBR (Open Thread Border Router), which can support both wifi and open-thread protocols and acts as a bridge between both.

      • In the last part of the diagram, there are two end devices connected to the Raspberry Pi. For our project, we will use two nRF5340 DK boards, which will resemble our matter light bulb and matter switch devices. These end devices will help us showcase the use of the matter protocol and its advantages.

    Matter Devices

      The diagram below shows a closer look at the Matter devices running on the Thread Network. We can see in addition to being connected to the Border Router, the matter devices are also connected to each other; creating a mesh network. What this means is that devices can communicate with one another on the network For instance, if a Matter-enabled light bulb and switch are on the same matter fabric and bound together, then we can manipulate the light bulb by toggling the switch. Essentially it allows us to indirectly interact with the devices we want; by using other devices.

    Functions

    Create an account and log in of the application website

    User can click on the signup button on top right to signup or log in

    Log out of the application on web browser

    User can click the logout button on the top right of the home page after they're signed in

    Able to commission matter devices into the network

    User can click the "Pair Devices" page on the top right and provide the necessary information for commission i.e. device type and ID

    Able to toggle the state of a device

    Toggle the state of a matter lightbulb by simply pressing the toggle button in the "View Devices" page

    Able to change the brightness of the light bulb

    Change the brightness by manipulating the brightness control lever in the "View Devices" page

    Able to bind two matter devices together

    Can bind two devices by pressing on the bind button which will allow the device with the controller role to manipulate the controlee device

    Able to control toggle the state of a bound device

    Can turn on and turn off the state of the controlee device from the controller device in the "View Devices" page

    Able to toggle state by physically interacting with the board and seeing reflected change on the website

    Can toggle state of a light by physically pressing button two on the NRF5340DK board

    View statuses of connected home systems in real-time

    User can click the "View Devices" page on the top right to see currently added devices to their account

    Tools

    nRF Connect For Desktop

    Multi-OS development tool to generate and debug code. Used to communicate with the nRF board. We plan on referring to the documentation from the Nordic website on how to utilize their nRF SDK

    Flask

    Utilized for the web application backend to enable users to form accounts, connect to specific devices, and will make the web application also process requests made to the end device as well as our Database

    Raspberry Pi’s

    We are taking advantage of Raspberry Pi’s simplicity and efficiency by running our Linux CHIP-Tool on one of them and the other will serve as the Open Thread Boarder Router.

    Nordic 5340DK

    We’re utilizing these Development Kits to emulate as a matter light bulb and light switch. We will refer to matter documentation on how to flash and properly commission these devices onto the thread network.
